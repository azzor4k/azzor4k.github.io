MCP Control Platform Architecture

Tài liệu mô tả cách xây dựng hệ thống quản lý và kiểm soát MCP Gateway cho AI tools truy cập các hệ thống nội bộ như SonarQube, GitLab, Jira, Database hoặc Cloud API.

1. MCP Overview

IDE / AI Client ↓ MCP Gateway ↓ Internal Services

MCP (Model Context Protocol) cho phép AI tool gọi các service nội bộ thông qua tool interface.

2. MCP Deployment Models

Model 1 — Shared Token

IDE ↓ MCP Server (shared token) ↓ Service API
  • Simple
  • Good for POC
  • No audit
  • Token leak risk

Model 2 — Per User Token

IDE ↓ MCP Server (user token) ↓ Service API
  • User audit
  • RBAC support
  • Token management required

Model 3 — MCP Gateway + API Proxy

IDE ↓ MCP Gateway ↓ API Proxy ↓ Auth / SSO ↓ Secret Manager ↓ Service API

Đây là kiến trúc production được khuyến nghị.

5. MCP Control Plane Architecture

Hệ thống quản lý MCP nên tách thành Control Plane và Data Plane.

Control Plane - Tool registry - Policy engine - RBAC - Audit logs ↓ MCP Gateway Runtime

6. Control Plane Components

Component Purpose
Tool Registry Store tool metadata
Policy Engine Control AI tool permissions
RBAC User access control
Audit Log Track AI tool usage

10. Threat Model (AI + MCP)

Threat Description Mitigation
Token leak Backend token exposed Store token in Vault
Prompt Injection AI tricked to call sensitive tools Tool permission control
Privilege escalation User access unauthorized resource RBAC + per-user token
API abuse AI generating too many calls Rate limiting
Data exfiltration AI leaking internal data Proxy filtering + audit logs
