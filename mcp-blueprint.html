MCP Deployment Blueprint (SecureScan Example)

Tài liệu mô tả các mô hình triển khai MCP server để AI / IDE truy cập hệ thống security nội bộ giả định tên SecureScan.

1. Tổng quan MCP

IDE / AI ↓ MCP Server ↓ Internal Service (SecureScan)

2. Phương án 1 — Shared Token MCP

IDE / AI ↓ MCP Server (shared token) ↓ SecureScan API

Ví dụ code

const token = process.env.SECURESCAN_TOKEN

server.tool(
"securescan_issues",
{ project: z.string() },

async ({project}) => {

const response = await fetch(
"https://securescan/api/issues?project=" + project,
{
headers:{
Authorization: `Bearer ${token}`
}
}
)

return response.json()

}
)

Ưu điểm

  • Triển khai nhanh
  • Phù hợp POC

Nhược điểm

  • Không audit user
  • Token leak → toàn hệ thống bị truy cập

3. Phương án 2 — Per User MCP

IDE / AI ↓ MCP Server (user token) ↓ SecureScan API 
server.tool(
"securescan_issues",
{ project: z.string() },

async ({project}, ctx) => {

const token = ctx.session.securescan_token

const response = await fetch(
"https://securescan/api/issues?project=" + project,
{
headers:{
Authorization: `Bearer ${token}`
}
}
)

return response.json()

}
)

4. Phương án 3 — MCP Gateway + API Proxy

IDE / AI ↓ MCP Gateway ↓ API Proxy ↓ Auth / SSO ↓ Secret Manager ↓ SecureScan API

Gateway code

server.tool(
"securescan_find_issues",
{ project: z.string() },

async ({ project }, ctx) => {

const response = await fetch(
`${process.env.API_PROXY}/securescan/issues?project=${project}`,
{
headers:{
Authorization:`Bearer ${ctx.session.access_token}`
}
}
)

return response.json()

}
)

IDE ↓ MCP Server ↓ LiteLLM Proxy ↓ SecureScan API

  litellm_params:
    api_base: https://securescan.internal/api

8. Threat Model (MCP + AI Access Internal Systems)

Threat Risk Mitigation
Token leakage AI tool lộ backend token Store token in Vault
Prompt Injection AI bị lừa gọi API nhạy cảm Tool permission control
Privilege escalation User truy cập project khác RBAC + per-user token
API abuse AI generate request spam Rate limiting
Data exfiltration AI leak dữ liệu internal Proxy filter + logging
